# 피란수도 부산 온톨로지 설계

> DH 이해하기 III · 워크숍 실습 예시 — *피란수도 부산 (1950–1953)*
> 커리큘럼 4단계(개체 탐색 → 클래스 → 속성 → 관계)를 따라 설계한 레퍼런스 문서.

온톨로지란 "우리가 다루는 세계에 **무엇이 있고(개체·클래스)**, 각 개체가 **어떤 성질을 가지며(속성)**, 개체들이 **서로 어떻게 이어지는지(관계)**"를 명시적으로 약속해 둔 것이다. 아래 4단계는 그 약속을 만드는 순서다.

---

## ① 개체 탐색 — 데이터에 실제로 등장하는 것들

세 데이터 파일(`places.csv`, `events.csv`, `triples.csv`)과 본문(`texts.txt`)에서 고유명사·사건·작품을 뽑아낸다.

| 유형 | 개체 |
|------|------|
| 인물 | 이중섭, 김환기, 윤이상, 김동리, 이승만 |
| 집단 행위자 | 대한민국정부, 피란민, 헌책방(상인) |
| 장소 | 부산, 영도, 국제시장, 40계단, 부산항, 자갈치시장, 광복동, 밀다원다방, 임시수도정부청사, 임시수도대통령관저, 보수동책방골목, 우암동소막마을, 아미동비석마을, 감천문화마을, 원산 |
| 사건 | 정부 임시수도 부산 이전, 1·4 후퇴, 부산정치파동, 발췌개헌, 국제시장 대화재, 정전협정 체결, 정부 서울 환도, 부산역전 대화재 |
| 작품 | 은지화, 밀다원시대 |

## ② 클래스 — 개체를 묶는 범주

커리큘럼의 핵심 4개 클래스를 사용하고, "집단 행위자"를 위해 `Agent`(행위자) 상위 클래스를 둔다.

```
Thing (최상위)
├── Agent  행위자 — 사건을 일으키거나 작품을 만드는 주체
│   ├── Person  개인          예) 이중섭, 이승만
│   └── Group   집단·기관      예) 대한민국정부, 피란민
├── Place   장소               예) 국제시장, 영도
├── Event   사건               예) 부산정치파동
└── Work    작품               예) 은지화, 밀다원시대
```

> **설계 판단:** 커리큘럼은 `Person / Place / Event / Work` 4개를 제시한다. 여기서는 `대한민국정부`·`피란민`처럼 개인이 아닌 주체가 있어 `Agent`를 상위에 두고 `Person`/`Group`으로 나눴다. 워크숍에서는 이를 그냥 `Person`으로 단순화해도 무방하다 — **정답은 하나가 아니며, 데이터를 설명하기에 적절한지가 기준이다.**

각 클래스가 데이터에서 맡는 시각화 역할:

| 클래스 | 주 데이터 | 시각화 |
|--------|----------|--------|
| Place | `places.csv` | 지도 (Leaflet) |
| Event | `events.csv` | 타임라인 (vis-timeline) |
| Agent/Work/Place | `triples.csv` | 관계망 (vis-network) |
| (본문) | `texts.txt` | 워드클라우드 |

## ③ 속성 — 각 개체가 자체적으로 갖는 성질 (data property)

`places.csv`·`events.csv`의 컬럼이 곧 속성이다. 속성은 **한 개체의 값**을 기술하며(관계와 달리 다른 개체를 가리키지 않음), 대개 시각화의 좌표·라벨·설명이 된다.

**Place 의 속성**

| 속성 | 예시 | 시각화 쓰임 |
|------|------|------------|
| `name` 이름 | 국제시장 | 마커 라벨 |
| `lat` 위도 | 35.0985 | 지도 좌표 |
| `lng` 경도 | 129.0286 | 지도 좌표 |
| `placeType` 장소유형 | 시장 / 섬 / 관청 … | 마커 색·아이콘 |
| `description` 설명 | 피란민 상권의 중심… | 팝업 본문 |

**Event 의 속성**

| 속성 | 예시 | 시각화 쓰임 |
|------|------|------------|
| `date` 일자 | 1953-01-30 | 타임라인 위치 |
| `title` 제목 | 국제시장 대화재 | 타임라인 라벨 |
| `atPlace` 발생장소 | 국제시장 | 지도-타임라인 연동 |
| `description` 설명 | 국제시장 일대를 태운 큰불 | 상세 패널 |

> **실무 권고:** 실제 프로젝트에서는 모든 개체에 `source`(출처) 속성을 추가하라. 이 실습 데이터는 교육용 근사값이다.

## ④ 관계 — 개체와 개체를 잇는 연결 (object property)

`triples.csv`의 `predicate`(서술어)가 관계다. 관계는 **주어(subject) → 서술어(predicate) → 목적어(object)** 형태의 트리플로, 지식그래프/관계망의 엣지가 된다.

관계를 의미별로 묶으면 다음과 같다.

**피란·이동** — 사람이 어디서 어디로 움직였나
| 관계 | 정의역(주어) | 치역(목적어) | 예 |
|------|------|------|----|
| `evacuatedFrom` 피란출발 | Agent | Place | 이중섭 → 원산 |
| `evacuatedTo` 피란도착 | Agent | Place | 김환기 → 부산 |
| `relocatedTo` 이전 | Group | Place | 대한민국정부 → 부산 |
| `stayedAt` 머묾 / `residedAt` 거주 | Agent | Place | 이중섭 → 영도 |
| `gatheredAt` 집결 / `settledAt` 정착 | Group | Place | 피란민 → 국제시장 |

**활동·창작** — 사람이 무엇을 했나
| 관계 | 정의역 | 치역 | 예 |
|------|------|------|----|
| `workedAt` 활동 / `frequented` 자주드나듦 | Agent | Place | 김동리 → 밀다원다방 |
| `created` 창작 / `wrote` 집필 | Agent | Work | 이중섭 → 은지화 |
| `usedAsOffice` 청사사용 | Group | Place | 대한민국정부 → 임시수도정부청사 |
| `formedAt` 형성 | Group | Place | 헌책방 → 보수동책방골목 |

**작품·장소 귀속** — 무엇이 어디에 속하나
| 관계 | 정의역 | 치역 | 예 |
|------|------|------|----|
| `createdAt` 제작장소 / `setIn` 배경 | Work | Place | 은지화 → 영도 |
| `locatedIn` 위치 | Place | Place | 밀다원다방 → 광복동 |

**사건 관여** — 누가/무엇이 사건에 얽혔나
| 관계 | 정의역 | 치역 | 예 |
|------|------|------|----|
| `involvedIn` 관여 | Agent | Event | 이승만 → 부산정치파동 |
| `partOf` 일부 | Event | Event | 발췌개헌 → 부산정치파동 |
| `occurredAt` 발생 | Event | Place | 국제시장대화재 → 국제시장 |

## 설계 검증 체크리스트

- [x] `triples.csv`의 모든 주어·목적어가 ①의 개체 목록에 존재하는가
- [x] 모든 개체가 ②의 클래스 중 하나에 배정되는가
- [x] `places.csv`·`events.csv`의 모든 컬럼이 ③의 속성으로 정의되는가
- [x] `triples.csv`의 모든 서술어가 ④에서 정의역·치역과 함께 정의되는가

이 4가지가 맞으면 데이터·설계·시각화가 어긋나지 않는다. 형식화된 스키마는 [`ontology.jsonld`](ontology.jsonld) 참고.
